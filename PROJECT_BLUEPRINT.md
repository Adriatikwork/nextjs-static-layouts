## Project Overview

**Project Name:** Fahrschule 06 (Driving School Website)  
**Type:** Static Business Landing Page  
**Live URL:** https://www.fahrschule06.ch  
**GitHub Repository:** https://github.com/Adriatik-Isufi/adriatik-isufi.github.io

### What This Project Does

- Modern, responsive single-page landing website for a local business
- Static site generation (SSG) for fast loading and free hosting on GitHub Pages
- Contact form with email integration
- Reviews/testimonials system with JSON data
- Instagram-style success stories viewer
- Video hero sections
- Mobile-first responsive design with custom sidebar navigation

## Project Architecture

### Static Export Configuration

The project is configured for **static export** (`output: 'export'`) in `next.config.mjs`:

```javascript
const nextConfig = {
  eslint: { ignoreDuringBuilds: true },
  typescript: { ignoreBuildErrors: true },
  images: { unoptimized: true },  // Required for static export
  output: 'export',               // Generates static HTML
  trailingSlash: true,            // Better for static hosting
  basePath: '',
  assetPrefix: '',
}
```

### App Router Structure

Uses Next.js 14+ App Router with the following structure:

```
app/
├── layout.tsx      # Root layout with metadata, fonts, SEO
├── page.tsx        # Main homepage (single-page app)
├── globals.css     # Global styles + Tailwind
├── impressum/
│   └── page.tsx    # Legal/Imprint page
└── sitemap.ts      # Dynamic sitemap generation
```

## GitHub & Deployment

### Repository Setup

**Repository:** GitHub Pages with custom domain  
**Branch:** `main`  
**Output:** Static files in `/out` directory

### GitHub Actions Workflow (`.github/workflows/deploy.yml`)

```yaml
name: Deploy Next.js to GitHub Pages

on:
  push:
    branches: [ main, master ]
  pull_request:
    branches: [ main, master ]

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '18'
          
      - name: Setup pnpm
        uses: pnpm/action-setup@v2
        with:
          version: 8
          run_install: false
          
      - name: Get pnpm store directory
        shell: bash
        run: echo "STORE_PATH=$(pnpm store path --silent)" >> $GITHUB_ENV
          
      - name: Setup pnpm cache
        uses: actions/cache@v3
        with:
          path: ${{ env.STORE_PATH }}
          key: ${{ runner.os }}-pnpm-store-${{ hashFiles('**/pnpm-lock.yaml') }}
          
      - name: Install dependencies
        run: pnpm install --no-frozen-lockfile
        
      - name: Build Next.js app
        run: pnpm run build
        env:
          NODE_ENV: production
          
      - name: Setup Pages
        uses: actions/configure-pages@v4
        
      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./out

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

### Custom Domain Setup

1. **CNAME file** in root: Contains `www.fahrschule06.ch`
2. **GitHub Settings:**
   - Go to Repository → Settings → Pages
   - Set custom domain
   - Enable HTTPS

### Deployment Flow

```
Push to main → GitHub Actions builds → Static files uploaded → GitHub Pages serves
```

## Step-by-Step Replication Guide

### Phase 1: Project Setup

```bash
# 1. Create new Next.js project with v0.dev or manually
npx create-next-app@latest my-business-site --typescript --tailwind --app

# 2. Install dependencies
cd my-business-site
pnpm add @emailjs/browser @lottiefiles/dotlottie-react lucide-react
pnpm add class-variance-authority clsx tailwind-merge next-themes

# 3. Install shadcn/ui
npx shadcn@latest init
npx shadcn@latest add button card badge input textarea

# 4. Configure for static export in next.config.mjs
```

### Phase 2: Structure Setup

```bash
# Create directories
mkdir -p components/ui data public/Success archive .github/workflows
```

### Phase 3: Copy Core Files

1. Copy configuration files:
   - `next.config.mjs`
   - `tailwind.config.ts`
   - `components.json`
   - `tsconfig.json`
   - `postcss.config.mjs`

2. Copy Docker files:
   - `Dockerfile`
   - `Dockerfile.dev`
   - `docker-compose.yml`
   - `.dockerignore`

3. Copy deployment files:
   - `.github/workflows/deploy.yml`
   - `.gitignore`

4. Copy PWA/SEO files:
   - `public/manifest.json`
   - `public/browserconfig.xml`
   - `public/robots.txt`
   - `CNAME` (for custom domain)

### Phase 4: Customize Content

1. Replace all brand colors (`#1351d8` → your color)
2. Update business information in:
   - `app/layout.tsx` (metadata, structured data)
   - `app/page.tsx` (all content)
   - `app/impressum/page.tsx`
   - `CNAME`

3. Replace assets:
   - Logo files
   - Hero images
   - Videos
   - Success photos

4. Update JSON data:
   - `data/reviews.json`
   - `data/stories.json`

### Phase 5: Setup Integrations

1. **EmailJS:** Create account, get credentials, update in `page.tsx`
2. **WhatsApp:** Update phone number in all links
3. **Social Media:** Update all social links
4. **Google Maps:** Update review URL

### Phase 6: Test & Deploy

```bash
# Local development
pnpm dev

# Build test
pnpm build

# Docker test
pnpm docker:dev

# Push to GitHub
git init
git add .
```

**Problem:** Build fails on GitHub Actions

## FAQ

**Q: Can I use this with npm instead of pnpm?**  
A: Yes, replace `pnpm` with `npm` in all commands. Update `package-lock.json` instead of `pnpm-lock.yaml`.

**Q: How do I change the primary color?**  
A: Find and replace `#1351d8` across all files. Update CSS variables in `globals.css` and `tailwind.config.ts`.

**Q: Can I add a blog section?**  
A: Yes, create `app/blog/` directory and add markdown files. Use a library like `gray-matter` for frontmatter parsing.

**Q: How do I add Google Analytics?**  
A: Add Google Analytics script to `app/layout.tsx` in the `<head>` section.

**Q: Can I use a different email service?**  
A: Yes, replace EmailJS with any email API (SendGrid, Mailgun, etc.) or use a backend API route.

**Q: How do I add more pages?**  
A: Create new folders in `app/` directory (e.g., `app/services/page.tsx`). Update navigation links.

**Q: Is this SEO-friendly?**  
A: Yes, includes metadata, structured data, sitemap, robots.txt, and semantic HTML.

**Q: Can I use this for e-commerce?**  
A: Not recommended for static export. Consider Vercel deployment with API routes for dynamic features.

**Q: How do I backup my site?**  
A: GitHub repository is your backup. Clone locally and backup `data/` and `public/` folders separately.

**Q: Can I use a different hosting service?**  
A: Yes, deploy `/out` folder to any static hosting (Netlify, Vercel, AWS S3, etc.).
